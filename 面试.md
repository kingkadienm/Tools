
# Android面试问答详解

## 1. RecyclerView相关

### 1.1 RecyclerView的优点和四级缓存

**RecyclerView优点：**
- 强制使用ViewHolder模式，性能更好
- 支持多种布局管理器（Linear、Grid、StaggeredGrid）
- 内置动画支持，增删改有流畅动画
- 解耦设计，LayoutManager、Adapter、ItemDecoration分离

**四级缓存机制：**
1. **mAttachedScrap**：屏幕内缓存，存储正在布局的ViewHolder
2. **mCachedViews**：屏幕外缓存，默认大小2，按position精确匹配
3. **ViewCacheExtension**：自定义缓存扩展
4. **RecycledViewPool**：类型缓存池，按viewType缓存，默认每种类型5个

### 1.2 局部刷新机制

**局部刷新优势：**
- 避免整个item重新绘制，性能更好
- 支持精确更新指定字段
- 减少不必要的UI刷新

**实现方式：**
```java
public class MyAdapter extends RecyclerView.Adapter<MyViewHolder> {
    
    @Override
    public void onBindViewHolder(@NonNull MyViewHolder holder, int position, @NonNull List<Object> payloads) {
        if (payloads.isEmpty()) {
            // 全量刷新
            onBindViewHolder(holder, position);
        } else {
            // 局部刷新
            for (Object payload : payloads) {
                if (payload instanceof String) {
                    String changeType = (String) payload;
                    switch (changeType) {
                        case "UPDATE_TITLE":
                            holder.titleView.setText(mData.get(position).getTitle());
                            break;
                        case "UPDATE_LIKE":
                            holder.likeView.setText(String.valueOf(mData.get(position).getLikeCount()));
                            break;
                    }
                }
            }
        }
    }
    
    // 触发局部刷新
    public void updateItemTitle(int position, String newTitle) {
        mData.get(position).setTitle(newTitle);
        notifyItemChanged(position, "UPDATE_TITLE");
    }
    
    public void updateLikeCount(int position, int likeCount) {
        mData.get(position).setLikeCount(likeCount);
        notifyItemChanged(position, "UPDATE_LIKE");
    }
}
```

### 1.3 DiffUtil高效更新

**DiffUtil优势：**
- 自动计算数据差异，精确更新
- 支持移动、插入、删除动画
- 后台线程计算，不阻塞UI

**基础使用：**
```java
public class ItemDiffCallback extends DiffUtil.Callback {
    private List<DataItem> oldList;
    private List<DataItem> newList;
    
    public ItemDiffCallback(List<DataItem> oldList, List<DataItem> newList) {
        this.oldList = oldList;
        this.newList = newList;
    }
    
    @Override
    public int getOldListSize() {
        return oldList.size();
    }
    
    @Override
    public int getNewListSize() {
        return newList.size();
    }
    
    @Override
    public boolean areItemsTheSame(int oldItemPosition, int newItemPosition) {
        // 判断是否为同一个item（通常比较ID）
        return oldList.get(oldItemPosition).getId() == newList.get(newItemPosition).getId();
    }
    
    @Override
    public boolean areContentsTheSame(int oldItemPosition, int newItemPosition) {
        // 判断内容是否相同
        DataItem oldItem = oldList.get(oldItemPosition);
        DataItem newItem = newList.get(newItemPosition);
        return oldItem.equals(newItem);
    }
    
    @Override
    public Object getChangePayload(int oldItemPosition, int newItemPosition) {
        // 返回变化的字段，用于局部刷新
        DataItem oldItem = oldList.get(oldItemPosition);
        DataItem newItem = newList.get(newItemPosition);
        
        Bundle payload = new Bundle();
        if (!oldItem.getTitle().equals(newItem.getTitle())) {
            payload.putString("title", newItem.getTitle());
        }
        if (oldItem.getLikeCount() != newItem.getLikeCount()) {
            payload.putInt("likeCount", newItem.getLikeCount());
        }
        
        return payload.isEmpty() ? null : payload;
    }
}

// 在Adapter中使用
public void updateData(List<DataItem> newData) {
    DiffUtil.Callback callback = new ItemDiffCallback(mData, newData);
    DiffUtil.DiffResult result = DiffUtil.calculateDiff(callback);
    
    mData.clear();
    mData.addAll(newData);
    result.dispatchUpdatesTo(this);
}
```

**ListAdapter简化使用：**

```java
public class MyListAdapter extends ListAdapter<DataItem, MyViewHolder> {

                return oldItem.getId() == newItem.getId();

                Bundle payload = new Bundle();
                if (!oldItem.getTitle().equals(newItem.getTitle())) {
                    payload.putString("UPDATE_TITLE", newItem.getTitle());
                }
                return payload.isEmpty() ? null : payload;
            }
        };

        if (payloads.isEmpty()) {
            super.onBindViewHolder(holder, position, payloads);

            DataItem item = getItem(position);
            for (Object payload : payloads) {
                if (payload instanceof Bundle) {
                    Bundle bundle = (Bundle) payload;
                    if (bundle.containsKey("UPDATE_TITLE")) {
                        holder.titleView.setText(bundle.getString("UPDATE_TITLE"));
                    }
                }
            }
        }
    }
}
```

## 2. 单例模式

### 2.1 单例中的无参构造存在的作用

**单例构造函数作用：**
- `private`修饰符防止外部new实例
- 构造逻辑简单，由类自己控制实例创建
- 无参只是最常见、最安全的形式

```java
public class Singleton {
    private static Singleton instance;
    
    // private构造函数防止外部实例化
    private Singleton() {
        // 初始化逻辑
    }
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

### 4.2 ASR数据包乱序处理

**数据包乱序处理方案：**
```java
public class AudioPacketManager {
    private final Map<Integer, AudioPacket> packetBuffer = new ConcurrentHashMap<>();
    private int expectedSequence = 0;
    
    public void handlePacket(AudioPacket packet) {
        packetBuffer.put(packet.getSequence(), packet);
        
        // 处理连续的包
        while (packetBuffer.containsKey(expectedSequence)) {
            AudioPacket orderedPacket = packetBuffer.remove(expectedSequence);
            processAudioData(orderedPacket.getData());
            expectedSequence++;
        }
        
        // 超时处理：清理过期包
        cleanExpiredPackets();
    }
    
    private void cleanExpiredPackets() {
        long currentTime = System.currentTimeMillis();
        packetBuffer.entrySet().removeIf(entry -> 
            currentTime - entry.getValue().getTimestamp() > TIMEOUT);
    }
}
```

**关键点：**
- 使用序列号标识数据包顺序
- 缓存乱序包，等待前序包到达
- 设置超时机制，避免无限等待

## 6. Java内部类

### 6.1 内部类与静态内部类区别

**区别对比：**
```java
public class OuterClass {
    private String outerField = "outer";
    private static String staticField = "static";
    
    // 非静态内部类
    class InnerClass {
        public void accessOuter() {
            System.out.println(outerField); // 可直接访问外部类实例成员
            System.out.println(OuterClass.this.outerField); // 显式访问
        }
    }
    
    // 静态内部类
    static class StaticInnerClass {
        public void accessOuter() {
            // System.out.println(outerField); // 编译错误！
            System.out.println(staticField); // 只能访问静态成员
            
            // 访问非静态成员需要外部类实例
            OuterClass outer = new OuterClass();
            System.out.println(outer.outerField);
        }
    }
}
```

**关键区别：**
- 非静态内部类持有外部类引用，可能导致内存泄漏
- 静态内部类不持有外部类引用，更安全
- 静态内部类访问外部类非静态成员需要实例
### 6.2 静态内部类访问非静态内部类

**静态内部类访问非静态内部类：**
```java
public class OuterClass {
    private String outerField = "outer";
    
    class InnerClass {
        private String innerField = "inner";
        
        public String getInnerField() {
            return innerField;
        }
    }
    
    static class StaticInnerClass {
        public void accessInnerClass() {
            // 1. 先创建外部类实例
            OuterClass outer = new OuterClass();
            
            // 2. 通过外部类实例创建内部类实例
            InnerClass inner = outer.new InnerClass();
            
            // 3. 访问内部类成员
            String value = inner.getInnerField();
            System.out.println(value);
        }
    }
}
```

**关键步骤：**
1. 静态内部类不能直接访问非静态内部类
2. 必须先创建外部类实例
3. 通过外部类实例创建内部类实例
4. 然后才能访问内部类成员
## 7. Handler内存管理

### 7.1 Handler内存泄漏处理

**Handler内存泄漏解决方案：**
```java
public class MainActivity extends Activity {
    private MyHandler mHandler;
    
    // 静态内部类 + 弱引用
    private static class MyHandler extends Handler {
        private final WeakReference<MainActivity> mActivity;
        
        public MyHandler(MainActivity activity) {
            mActivity = new WeakReference<>(activity);
        }
        
        @Override
        public void handleMessage(Message msg) {
            MainActivity activity = mActivity.get();
            if (activity != null) {
                // 处理消息，activity可能为null需要判断
                activity.updateUI();
            }
        }
    }
    
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        mHandler = new MyHandler(this); // 传入Activity引用
    }
    
    @Override
    protected void onDestroy() {
        super.onDestroy();
        mHandler.removeCallbacksAndMessages(null); // 清理消息队列
    }
}
```

**如何拿到Activity：**
- 通过构造函数传入Activity引用
- 使用WeakReference避免强引用导致内存泄漏
- 在使用前检查Activity是否还存在

## 8. 图像处理

### 8.1 帧率识别优化

**帧率识别优化：**

**为什么选择15帧：**
- 平衡识别准确率和性能
- 10帧数据量不足，准确率低
- 20帧处理延迟高，实时性差
- 15帧是经验最优值

**性能优化处理：**
```java
public class FrameProcessor {
    private static final int TARGET_FPS = 15;
    private static final int FRAME_INTERVAL = 1000 / TARGET_FPS; // 66ms
    
    private long lastProcessTime = 0;
    private ExecutorService executor = Executors.newSingleThreadExecutor();
    
    public void processFrame(byte[] frameData) {
        long currentTime = System.currentTimeMillis();
        if (currentTime - lastProcessTime < FRAME_INTERVAL) {
            return; // 跳过帧，控制帧率
        }
        
        lastProcessTime = currentTime;
        
        // 异步处理，避免阻塞主线程
        executor.execute(() -> {
            // 图像预处理：缩放、裁剪
            Bitmap scaledBitmap = scaleBitmap(frameData, 320, 240);
            // 发送到识别服务
            sendToRecognition(scaledBitmap);
        });
    }
}
```

**内存占用计算：**
- 原始帧（1920x1080 ARGB_8888）：8.3MB
- 压缩后（320x240 RGB_565）：150KB
- 15帧总内存：约2.25MB
### 8.2 Bitmap内存计算

**Bitmap内存计算公式：**
```
内存大小 = 宽 × 高 × 每像素字节数
```

**不同格式的字节数：**
- **ARGB_8888**：4字节/像素（Alpha + RGB各8位）
- **RGB_565**：2字节/像素（RGB分别5、6、5位）
- **ARGB_4444**：2字节/像素（ARGB各4位）
- **ALPHA_8**：1字节/像素（只有透明度）

**计算示例：**
```java
// 1920x1080的图片
int width = 1920;
int height = 1080;

// ARGB_8888格式
long memoryARGB8888 = width * height * 4; // 8,294,400 bytes ≈ 8MB

// RGB_565格式  
long memoryRGB565 = width * height * 2;   // 4,147,200 bytes ≈ 4MB
```

**影响因素：**
- 图片尺寸（宽×高）
- 色彩格式（Config）
- 密度缩放（inDensity/inTargetDensity）

**优化建议：**
- 使用RGB_565减少内存占用
- 及时回收：bitmap.recycle()
- 使用BitmapFactory.Options控制采样率

### 8.4 YUV格式详解

**YUV格式详解：**

**YUV含义：**
- **Y**：亮度分量（Luminance），灰度信息
- **U**：蓝色色度分量（Cb）
- **V**：红色色度分量（Cr）

**常见YUV格式：**

**1. YUV420系列：**
- **YUV420P（I420）**：Y、U、V分别存储，U、V采样率为Y的1/4
- **YUV420SP（NV21）**：Y单独存储，UV交错存储，Android Camera默认格式
- **YUV420SP（NV12）**：Y单独存储，VU交错存储

**2. YUV422系列：**
- **YUYV（YUY2）**：Y、U、Y、V交错存储
- **UYVY**：U、Y、V、Y交错存储

**存储示例（NV21）：**
```
YYYY YYYY    <- Y平面（亮度）
YYYY YYYY
VUVU         <- UV平面（色度交错）
```

**摄像头输出格式：**
- Android Camera默认：**NV21**
- Camera2 API常用：**YUV_420_888**
- 人脸SDK通常接收：**NV21**或**YUV420P**

**格式转换：**
```java
public byte[] nv21ToYuv420p(byte[] nv21, int width, int height) {
    int ySize = width * height;
    int uvSize = ySize / 4;
    
    byte[] yuv420p = new byte[ySize + uvSize * 2];
    
    // 复制Y分量
    System.arraycopy(nv21, 0, yuv420p, 0, ySize);
    
    // 分离UV分量
    for (int i = 0; i < uvSize; i++) {
        yuv420p[ySize + i] = nv21[ySize + i * 2 + 1]; // U
        yuv420p[ySize + uvSize + i] = nv21[ySize + i * 2]; // V
    }
    
    return yuv420p;
}
```
## 9. Service管理

### 9.1 Service启动方式对比

**区别对比：**

### 10.3 音频焦点管理

**音频焦点管理方案：**

```java
public class TTSManager implements AudioManager.OnAudioFocusChangeListener {
    private AudioManager mAudioManager;
    private TextToSpeech mTTS;
    private boolean mHasAudioFocus = false;
    
    public void playTTS(String text) {
        // 1. 请求音频焦点
        int result = mAudioManager.requestAudioFocus(
            this,
            AudioManager.STREAM_MUSIC,
            AudioManager.AUDIOFOCUS_GAIN_TRANSIENT_MAY_DUCK // 短暂获取焦点，允许其他音频降低音量
        );
        
        if (result == AudioManager.AUDIOFOCUS_REQUEST_GRANTED) {
            mHasAudioFocus = true;
            mTTS.speak(text, TextToSpeech.QUEUE_FLUSH, null, null);
        }
    }
    
    @Override
    public void onAudioFocusChange(int focusChange) {
        switch (focusChange) {
            case AudioManager.AUDIOFOCUS_GAIN:
                // 重新获得焦点，恢复播放
                mHasAudioFocus = true;
                if (mTTS != null) {
                    // 恢复TTS播放
                }
                break;
                
            case AudioManager.AUDIOFOCUS_LOSS:
                // 永久失去焦点，停止播放
                mHasAudioFocus = false;
                if (mTTS != null) {
                    mTTS.stop();
                }
                mAudioManager.abandonAudioFocus(this);
                break;
                
            case AudioManager.AUDIOFOCUS_LOSS_TRANSIENT:
                // 暂时失去焦点，暂停播放
                mHasAudioFocus = false;
                if (mTTS != null) {
                    mTTS.stop();
                }
                break;
                
            case AudioManager.AUDIOFOCUS_LOSS_TRANSIENT_CAN_DUCK:
                // 失去焦点但可以降低音量继续播放
                // TTS通常选择暂停，让音乐继续
                if (mTTS != null) {
                    mTTS.stop();
                }
                break;
        }
    }
    
    public void release() {
        if (mHasAudioFocus) {
            mAudioManager.abandonAudioFocus(this);
        }
        if (mTTS != null) {
            mTTS.shutdown();
        }
    }
}
```

**焦点类型选择：**
- **AUDIOFOCUS_GAIN**：长期占用（音乐播放）
- **AUDIOFOCUS_GAIN_TRANSIENT**：短期占用（通话）
- **AUDIOFOCUS_GAIN_TRANSIENT_MAY_DUCK**：短期占用，允许其他音频降音量（TTS、导航）

**处理策略：**
1. TTS播放前请求短暂焦点
2. 音乐应用收到焦点变化时降低音量或暂停
3. TTS播放完成后释放焦点
4. 音乐应用重新获得焦点后恢复播放

**内存泄漏 vs 内存溢出：**

**内存泄漏（Memory Leak）：**
- 对象无法被GC回收，内存逐渐累积
- 常见原因：静态引用、监听器未注销、Handler持有Activity引用

**内存溢出（OOM）：**
- 申请内存超过可用内存限制
- 常见原因：大图片加载、递归调用、内存泄漏累积

```java
public class ImageLoader {
    public static Bitmap loadBitmap(String path, int reqWidth, int reqHeight) {
        BitmapFactory.Options options = new BitmapFactory.Options();
        options.inJustDecodeBounds = true; // 只获取尺寸，不加载到内存
        BitmapFactory.decodeFile(path, options);
        
        // 计算采样率
        options.inSampleSize = calculateInSampleSize(options, reqWidth, reqHeight);
        options.inJustDecodeBounds = false;
        options.inPreferredConfig = Bitmap.Config.RGB_565; // 减少内存占用
        
        return BitmapFactory.decodeFile(path, options);
    }
    
    private static int calculateInSampleSize(BitmapFactory.Options options, 
                                           int reqWidth, int reqHeight) {
        int height = options.outHeight;
        int width = options.outWidth;
        int inSampleSize = 1;
        
        if (height > reqHeight || width > reqWidth) {
            int halfHeight = height / 2;
            int halfWidth = width / 2;
            
            while ((halfHeight / inSampleSize) >= reqHeight
                    && (halfWidth / inSampleSize) >= reqWidth) {
                inSampleSize *= 2;
            }
        }
        return inSampleSize;
    }
}
```
